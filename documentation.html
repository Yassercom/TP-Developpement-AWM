Documentation du Projet CRUD Full-Stack



Développement étape par étape d'une application complète avec React, Express et PostgreSQL



Encadré par : Pr Mohamed EL FAROUKI

Vue d'ensemble du projet

Ce projet est une application CRUD (Create, Read, Update, Delete) complète qui démontre l'intégration de technologies modernes du développement web. L'application permet aux utilisateurs de s'inscrire, se connecter, et gérer leurs items personnels avec toutes les opérations CRUD.

Frontend

Développé avec React.js pour une interface utilisateur dynamique et réactive.

React Context API React Router
Backend

Construit avec Express.js pour fournir une API RESTful performante.

Express Node.js JWT
Base de données

PostgreSQL pour un stockage relationnel robuste et fiable.

PostgreSQL

Structure du projet

Notre application suit une architecture moderne avec une séparation claire entre le frontend et le backend.

/TP-Developpement-AWM/backend/controllers - Logique métier
/middleware - Middleware pour l'authentification
/models - Modèles de données
/routes - Routes API
.env - Variables d'environnement
db.js - Configuration de la base de données
server.js - Point d'entrée du serveur
/frontend/public - Fichiers statiques
/src/components - Composants React réutilisables
/context - Contextes React pour la gestion d'état
/pages - Pages de l'application
/utils - Utilitaires

Étapes de développement

1 Configuration du projet

Nous avons commencé par créer la structure du projet avec des dossiers séparés pour le frontend et le backend, et mis en place les fichiers de configuration de base.

La bonne organisation du projet dès le départ facilite la maintenance et les futures améliorations.

2 Configuration de la base de données

Nous avons configuré PostgreSQL et créé la base de données crud_app avec les tables nécessaires.

-- Create users table CREATE TABLE IF NOT EXISTS users ( id SERIAL PRIMARY KEY, name VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE NOT NULL, password VARCHAR(100) NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );
-- Create items table CREATE TABLE IF NOT EXISTS items ( id SERIAL PRIMARY KEY, title VARCHAR(100) NOT NULL, description TEXT, user_id INTEGER REFERENCES users(id) ON DELETE CASCADE, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );

3 Développement du backend

Nous avons créé un serveur Express avec des routes RESTful pour gérer l'authentification et les opérations CRUD sur les items.

  • Authentification: Implémentation JWT pour sécuriser l'application
  • Routes API: Création de routes pour les utilisateurs et les items
  • Middleware: Protection des routes sensibles
// Routes d'authentification : app.use('/api/auth', require('./routes/auth'));
// des items (protégées) app.use('/api/items', require('./routes/items'));

4 Développement du frontend

Nous avons construit l'interface utilisateur avec React, y compris des pages pour l'inscription, la connexion et le tableau de bord.

  • Composants: Création de composants réutilisables
  • Context API: Gestion de l'état global de l'application
  • Routes protégées: Contrôle d'accès basé sur l'authentification

5 Amélioration du tableau de bord

Nous avons enrichi le tableau de bord avec des fonctionnalités avancées.

  • Statistiques: Affichage de données récapitulatives
  • Recherche: Filtrage des items en temps réel
  • UI améliorée: Interface utilisateur intuitive et esthétique

6 Tests et débogage

Nous avons résolu plusieurs problèmes et optimisé l'application.

Nous avons notamment résolu un problème où les tables n'étaient pas créées dans la bonne base de données, ce qui causait des erreurs 500 lors des requêtes API.

Installation de l'Application

Prérequis

  • Node.js (version 14 ou supérieure)
  • NPM (gestionnaire de paquets Node.js)
  • PostgreSQL (version 12 ou supérieure)

Installation du Backend

1. Cloner le repository :

git clone https://github.com/Yassercom/TP-Developpement-AWM.git

2. Se positionner dans le dossier backend :

cd TP-Developpement-AWM/backend

3. Installer les dépendances :

npm install

4. Configurer les variables d'environnement :

cp .env.example .env

Modifier le fichier .env avec vos informations de configuration

5. Créer la base de données PostgreSQL :

createdb crud_app

6. Lancer le serveur :

npm start

Installation du Frontend

1. Se positionner dans le dossier frontend :

cd TP-Developpement-AWM/frontend

2. Installer les dépendances :

npm install

3. Lancer l'application React :

npm start

Configuration de l'Environnement

Backend (.env)

  • PORT=5000
  • DATABASE_URL=postgresql://postgres:your_password@localhost:5432/crud_app
  • JWT_SECRET=votre_clé_secrète

Frontend (package.json)

Assurez-vous que le proxy pointe vers le backend :

"proxy": "http://localhost:5000"
